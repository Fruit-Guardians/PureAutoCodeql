## 1. 模块概览

### 1.1 项目定位

Config 模块是 PureAutoCodeQL 项目的**统一配置管理中心**，负责管理所有 LLM（大语言模型）服务商的配置、API Key、模型选择等核心参数。

### 1.2 设计目标

- **零代码配置**：用户仅需修改 TOML 配置文件，无需修改源代码
- **多服务商支持**：内置 5 个主流服务商，支持无限自定义服务商
- **完全向后兼容**：保留旧版所有 API，现有代码无需修改
- **安全优先**：API Keys 通过 `.gitignore` 保护，支持环境变量
- **开发友好**：清晰的模块划分，丰富的 CLI 工具

### 1.3 技术特性

| 特性 | 说明 |
|------|------|
| 配置文件格式 | TOML（人类友好的配置语言） |
| 服务商协议 | OpenAI 兼容 API |
| 配置优先级 | CLI > 环境变量 > keys.toml > 默认值 |
| 验证机制 | 网络可达性检查 + 实际 API 调用验证 |
| 展示层 | Rich 库实现的终端美化输出 |

### 3.2 服务商注册中心 (ProviderRegistry)

```python
class ProviderRegistry:
    """服务商注册中心 (单例模式)"""
    _providers: dict[str, ProviderConfig] = {}  # 服务商存储
    _keys_config: dict = {}                     # keys.toml 配置缓存
```

**核心方法：**

1. `register(config: ProviderConfig)` - 注册服务商
2. `get(name: str)` - 获取服务商
3. `list_all()` - 列出所有服务商
4. `list_available()` - 列出可用服务商（已配置且可达）
5. `exists(name: str)` - 检查是否存在
6. `load_keys_toml(filepath)` - 加载配置文件
7. `get_default_provider()` - 获取默认服务商
8. `get_default_models()` - 获取默认模型

**加载流程：**

```python
def load_keys_toml(cls, filepath):
    # 1. 读取 TOML 文件
    with open(filepath, 'rb') as f:
        cls._keys_config = tomllib.load(f)
    
    # 2. 为内置服务商设置 API Keys
    builtin_keys = cls._keys_config.get('builtin_keys', {})
    for name, key in builtin_keys.items():
        if key and name in cls._providers:
            cls._providers[name]._dev_key = key
    
    # 3. 注册自定义服务商
    custom_providers = cls._keys_config.get('custom_providers', [])
    for provider_data in custom_providers:
        cls._register_custom_from_toml(provider_data)
```

### 3.4 核心配置函数 (core.py)

#### get_llm_config()

```python
def get_llm_config(
    role: LLMRole,                      # 必填：THINK 或 CHAT
    provider_name: Optional[str] = None,  # 可选：服务商名称
    model_name: Optional[str] = None,     # 可选：模型名称
    api_key: Optional[str] = None,        # 可选：API Key
    base_url: Optional[str] = None,       # 可选：Base URL
    auto_fallback: bool = False           # 可选：自动回退
) -> LLMConfig:
    """根据角色获取 LLM 配置"""
```

**执行流程：**

```python
# 1. 确定服务商
if provider_name:
    target_provider = ProviderRegistry.get(provider_name)
else:
    # 从 keys.toml 或环境变量读取默认服务商
    default_name = ProviderRegistry.get_default_provider()
    target_provider = ProviderRegistry.get(default_name)

# 2. 自动回退 (如果启用)
if auto_fallback and not target_provider.is_configured():
    available_providers = ProviderRegistry.list_available()
    if available_providers:
        target_provider = available_providers[0]

# 3. 确定 API Key 和 Base URL
final_api_key = api_key or target_provider.get_api_key()
final_base_url = base_url or target_provider.get_base_url()

# 4. 确定模型
if model_name:
    final_model = model_name
elif role == LLMRole.THINK:
    final_model = default_think or target_provider.default_think_model
else:
    final_model = default_chat or target_provider.default_chat_model

# 5. 返回 LLMConfig
return LLMConfig(
    model=final_model,
    api_key=final_api_key,
    base_url=final_base_url,
    provider=target_provider.name,
    # ... 其他默认参数
)
```

**实现关键代码：**

```python
# ProviderConfig.get_api_key()
def get_api_key(self) -> str:
    # 1. 专属环境变量
    for env_name in self.env_keys:
        if os.getenv(env_name):
            return os.getenv(env_name)
    
    # 2. 通用环境变量
    for env_name in ["OPENAI_API_KEY", "API_KEY"]:
        if os.getenv(env_name):
            return os.getenv(env_name)
    
    # 3. keys.toml (由 ProviderRegistry 设置)
    if hasattr(self, '_dev_key'):
        return self._dev_key
    
    return ""
```

### 4.3 API 验证机制

**两级验证：**

1. **快速检查** - 网络可达性
   ```python
   def is_reachable(self, timeout: float = 2.0) -> bool:
       try:
           req = Request(self.get_base_url(), method="GET")
           with urlopen(req, timeout=timeout) as resp:
               return True
       except HTTPError:
           return True  # HTTP 错误说明服务可达
       except (URLError, Exception):
           return False
   ```

2. **深度检查** - 实际 API 调用
   ```python
   def validate_api_key(self) -> tuple[bool, str]:
       # 方法 1: 尝试 /models 接口
       url = f"{self.get_base_url()}/models"
       response = client.get(url, headers=headers)
       
       if response.status_code == 200:
           return True, "验证成功"
       elif response.status_code == 401:
           return False, "API Key无效"
       
       # 方法 2: 尝试最小 chat 请求
       test_data = {
           "model": self.default_chat_model,
           "messages": [{"role": "user", "content": "hi"}],
           "max_tokens": 1
       }
       response = client.post(f"{base_url}/chat/completions", ...)
   ```

**状态标记系统：**

```python
def get_status(self, validate_key: bool = False) -> tuple[str, str]:
    has_key = self.is_configured()
    
    if not has_key:
        return ("⚠️", "API Key 缺失")
    
    if not validate_key:
        # 快速检查
        reachable = self.is_reachable()
        return ("✅", "已配置") if reachable else ("❌", "不可达")
    else:
        # 深度检查
        is_valid, error_msg = self.validate_api_key()
        return ("✅", "已验证") if is_valid else ("❌", error_msg)
```

```python
    return get_llm_config(role, provider_name=provider_name)

def list_available_providers() -> list[dict]:
    """旧版函数：返回字典列表格式"""
    providers_info = []
    for provider in ProviderRegistry.list_all():
        provider_info = {
            "name": provider.name,
            "display_name": provider.display_name,
            "think_model": provider.default_think_model,
            "chat_model": provider.default_chat_model,
            # ... 其他字段
        }
        providers_info.append(provider_info)
    return providers_info
```

## 5. 配置系统

### 5.1 配置文件规范

**文件类型：**
- `keys.example.toml` - 提交到仓库的模板文件
- `keys.toml` - 用户实际使用的配置文件（被 .gitignore 忽略）

**TOML 语法规范：**

```toml
# 1. 字典 (表)
[section_name]
key = "value"

# 2. 数组 (表数组)
[[array_name]]
key1 = "value1"
key2 = "value2"

[[array_name]]  # 第二个元素
key1 = "value3"

# 3. 嵌套字典
[section.subsection]
key = "value"
```

### 5.2 内置服务商配置

| 服务商 | 环境变量 | 推理模型 | 对话模型 |

| DeepSeek | `DEEPSEEK_API_KEY` | deepseek-reasoner | deepseek-chat |
| SiliconFlow | `SILICONFLOW_API_KEY` | deepseek-ai/DeepSeek-R1 | Pro/deepseek-ai/DeepSeek-V3.2-Exp |
| 智谱GLM | `ZHIPU_API_KEY` | glm-4.6 | glm-4.6 |

### 5.3 自定义服务商配置

**完整配置示例：**

```toml
[[custom_providers]]
name = "my_ollama"                  # 必填：唯一标识符
display_name = "本地 Ollama"         # 必填：显示名称
api_key = "ollama"                  # 必填：API Key
base_url = "http://localhost:11434/v1"  # 必填：API 地址
think_model = "deepseek-r1:latest"  # 必填：推理模型
chat_model = "qwen2.5:latest"       # 必填：对话模型
description = "本地运行的 Ollama"    # 可选：描述
available_models = [                # 可选：模型列表
    "deepseek-r1:latest",
    "qwen2.5:latest",
]

# 自定义参数（保留扩展性）
[custom_providers.custom_params]
max_retries = 5
timeout = 60
```

**字段说明：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| name | string | ✅ | 内部标识符（小写字母、数字、下划线） |
| display_name | string | ✅ | 用户友好的显示名称 |
| api_key | string | ✅ | API Key |
| base_url | string | ✅ | API 基础地址 (需包含协议) |
| think_model | string | ✅ | 推理模型名称 |
| chat_model | string | ✅ | 对话模型名称 |
| description | string | ❌ | 服务商描述 |
| available_models | array | ❌ | 可用模型列表 |
| custom_params | dict | ❌ | 自定义参数 |

### 5.4 全局设置

```toml
[settings]
default_provider = "siliconflow"              # 默认服务商

```

**作用：**
- `default_provider` - 当命令行未指定 `--provider` 时使用
- `default_think_model` - 覆盖服务商的默认推理模型
- `default_chat_model` - 覆盖服务商的默认对话模型
