#### 新增方法1：`_extract_unique_files`
```python
def _extract_unique_files(self, nodes: List[Dict]) -> List[str]:
    """从节点列表中提取唯一的文件路径。"""
    files = set()
    for node in nodes:
        file_path = node.get('file', '') or node.get('file_path', '')
        if file_path:
            files.add(file_path)
    return list(files)
```

#### 新增方法2：`_read_multiple_source_files`
```python
def _read_multiple_source_files(self, file_paths: List[str], max_files: int = 5) -> str:
    """读取多个源文件并格式化为易读的文本。

    - 最多读取5个文件（防止过大）
    - 单文件最多5000行（防止超出上下文）

    - 格式化输出（带文件名标记）
```

## 📊 完整工作流程

```
用户运行Python/C++项目分析
    ↓
CodeQL查询执行 → 结果为空
    ↓
【层次1】触发断流点查找（因为语言在列表中）
    ↓
生成双向可达性查询
    ↓
执行断流点查询
    ↓
【层次3】代码主动操作：
  1. 从查询结果提取Source/Sink文件路径
  2. 读取这些文件的完整内容（最多5个，每个最多5000行）
  3. 格式化为markdown代码块
    ↓
【层次2+3】构建增强提示词：
  - CodeQL查询结果
  - Source/Sink节点信息
  - **预加载的完整源码** ← 代码注入
  - 语言特定的分析指导 ← 提示词引导
    ↓
LLM深度分析：
  - 阅读完整源码（已经在提示词中）
  - 理解数据流路径
  - 识别具体的断流函数（如json.loads, memcpy）
    ↓
【层次2】LLM生成修复条件：
  - 基于语言专用提示词
  - 使用正确的CodeQL语法
  - 精确匹配断流函数
    ↓
将isAdditionalFlowStep条件插入原始查询
    ↓
重新执行查询
    ↓
成功 ✅ 或 重试（最多3次）
```

## 🛡️ 安全和限制

### 文件读取限制
```python
max_files = 5           # 最多5个文件
max_lines = 5000        # 单文件最多5000行
```

**总token估算**：
```
5文件 × 5000行 × 80字符/行 ≈ 2M字符 ≈ 500K tokens
```

**LLM上下文窗口**：100K-200K tokens（通常够用）

### 错误处理
- ✅ 文件不存在 → 友好提示
- ✅ 文件过大 → 自动截断并标注
- ✅ 编码错误 → 忽略错误继续（`errors='ignore'`）
- ✅ 读取失败 → 降级为传统断流查找
