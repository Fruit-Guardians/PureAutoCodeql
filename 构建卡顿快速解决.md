# CodeQL 构建看起来卡住了？不用担心！

## 🔍 快速判断

如果你看到：
```
[100%] Built target lib2033
```

然后就没有输出了，**这是正常的！** 

## ⏱️ 正在发生什么

CodeQL 正在执行 `database finalize` 步骤：
- 压缩源代码（生成 src.zip）
- 创建数据库索引
- 完成最终处理

**这个过程需要 5-15 分钟**，但没有进度输出。

## ✅ 确认没有真的卡死

### 方法 1：使用监控脚本（推荐）

打开新终端，运行：
```bash
python monitor_build.py CVE-2018-14618
```

会显示实时进度：
- 数据库大小
- CPU/内存使用
- 最新日志

### 方法 2：查看 Docker 状态

```bash
docker stats
```

如果 CPU 和内存使用率较高，说明正在工作。

### 方法 3：检查文件变化

```bash
# Windows PowerShell
Get-ChildItem projects\<case_id>\db\cpp -Recurse | Measure-Object -Property Length -Sum

# Linux/Mac
watch -n 5 "du -sh projects/<case_id>/db/cpp"
```

如果目录大小在增长，说明正常。

## 📝 如何使用新版本

### 1. 重新构建 Docker 镜像

```bash
cd docker
docker build -t codeql-builder:latest .
```

### 2. 运行构建

```bash
uv run Analyze.py --case "你的项目路径"
```

### 3. 在另一个终端监控进度

```bash
python monitor_build.py <case_id>
```

## ⚠️ 真的卡死了？

如果超过 30 分钟都没有进展，并且：
- Docker CPU 使用率为 0%
- 数据库大小不再增长
- 没有任何日志输出

可能真的卡死了。解决方法：

```bash
# 1. 强制停止
docker kill $(docker ps -q --filter ancestor=codeql-builder)

# 2. 清理数据库
rm -rf projects/<case_id>/db

# 3. 重新开始
uv run Analyze.py --case "..." --overwrite
```
